## Interpretation

- **loose_only_paraphrase** (~45% of failures): the loose token metric already
  passes; only the brittle strict substring misses. These are fixture-phrasing
  debt — candidates for `expected_alternates` additions, NOT retrieval bugs.
  They should be corrected case-by-case with the current authoritative
  phrasing as evidence, never bulk-copied from retrieved snippets (that would
  encode current output as expectation).
- **phrase_live_ranking_miss**: the expected phrase exists verbatim in live
  canonical/distilled files but the doc missed top-5. This is the genuine
  retrieval-quality frontier (mostly distilled docs without a vector index
  losing keyword contests to canonical, per the Contract 9 gateway-port RCA).
- **stale_expected_phrase** (97 on extended): fixture expects truth that no
  longer exists anywhere in live or archived knowledge. Concentrated in the
  extended track — fixture review debt, tracked here rather than silently
  rewritten.
- **zero_results**: strict governance surfaces (live-state suppression et al.)
  returning empty by design for trend probes phrased as status queries.
- **archived_expected_source flag**: eval fixtures predate the canonical
  archive sweeps; successor matching in `eval_compare._expected_hit` already
  grants source credit when a live successor surfaces, so this flag alone is
  not a failure cause — it marks where fixtures point at retired paths.
