## 1. 시작하기

### 옵시디언이란?
옵시디언은 **로컬 우선(local-first)** 마크다운 노트 앱입니다. 모든 파일이 컴퓨터에 `.md` 파일로 저장됩니다. 데이터는 온전히 여러분의 것이며, 클라우드에 종속되지 않습니다.

### 핵심 개념
- **볼트(Vault)**: 옵시디언이 작업 공간으로 사용하는 컴퓨터의 폴더. 모든 노트가 여기에 저장됩니다.
- **노트(Note)**: 하나의 마크다운(`.md`) 파일입니다.
- **링크(Link)**: `[[이중 괄호]]`를 사용해 노트 간 연결을 만듭니다.

### 첫 번째 볼트 만들기
1. 옵시디언을 실행합니다
2. **"새 볼트 만들기(Create new vault)"**를 클릭합니다
3. 이름을 입력합니다 (예: `나의 지식 베이스`)
4. 컴퓨터에서 저장 위치를 선택합니다
5. **만들기(Create)**를 클릭합니다

### 첫 번째 노트 만들기
1. 왼쪽 사이드바에서 **"새 노트"** 아이콘(📄)을 클릭하거나, `Ctrl/Cmd + N`을 누릅니다
2. 상단에 제목을 입력합니다
3. 아래에 내용을 작성합니다

**예시:**
```markdown
# 나의 첫 번째 노트

옵시디언 볼트에 오신 것을 환영합니다! 여기에 모든 지식을 저장할 것입니다.
```

## 2. 기본 편집 & 마크다운

옵시디언은 **마크다운(Markdown)**을 사용하여 서식을 지정합니다.

### 제목(Headings)
```markdown
# 제목 1
## 제목 2
### 제목 3
#### 제목 4
##### 제목 5
###### 제목 6
```

### 텍스트 서식
```markdown
**굵은 글씨**
*기울임 글씨*
~~취소선~~
==형광펜 효과==
**_굵은 기울임_**
```

**결과:**
- **굵은 글씨**
- *기울임 글씨*
- ~~취소선~~
- ==형광펜 효과==
- ***굵은 기울임***

### 목록

**순서 없는 목록:**
```markdown
- 항목 1
- 항목 2
  - 하위 항목 2a
  - 하위 항목 2b
```

**순서 있는 목록:**
```markdown
1. 첫 번째 단계
2. 두 번째 단계
3. 세 번째 단계
```

**체크리스트(할 일 목록):**
```markdown
- [x] 완료한 작업
- [ ] 미완료 작업
- [ ] 해야 할 다른 작업
```

### 인용문
```markdown
> 이것은 인용문입니다.
> 여러 줄에 걸칠 수 있습니다.
>
> > 중첩도 가능합니다.
```

### 코드

**인라인 코드:**
```markdown
파이썬에서 `print()` 함수를 사용합니다.
```

**코드 블록:**
````markdown
```python
def hello():
    print("안녕하세요, 옵시디언!")
```
````

### 구분선
```markdown
---
```

### 표(테이블)
```markdown
| 기능      | 상태    | 비고           |
|-----------|---------|---------------|
| 마크다운   | ✅ 완료 | 기본 지원      |
| 링크      | ✅ 완료 | 위키 스타일     |
| 그래프 뷰  | ✅ 완료 | 인터랙티브     |
```

### 각주
```markdown
여기에 각주가 있는 문장입니다.[^1]

[^1]: 이것이 각주의 내용입니다.
```

### 수학(LaTeX)

**인라인 수식:**
```markdown
공식은 $E = mc^2$입니다.
```

**블록 수식:**
```markdown
$$
\sum_{i=1}^{n} x_i = x_1 + x_2 + \cdots + x_n
$$
```

### 주석(숨겨진 텍스트)
```markdown
%%이 텍스트는 미리보기 모드에서 표시되지 않습니다%%
```

## 3. 링크 & 백링크

링크는 옵시디언의 **핵심 기능**입니다. 노트 간 연결을 만들어줍니다.

### 내부 링크
```markdown
[[노트 이름]]
```
**예시:** `[[나의 첫 번째 노트]]`는 "나의 첫 번째 노트" 파일로의 링크를 만듭니다.

### 표시 텍스트가 다른 링크
```markdown
[[노트 이름|표시할 텍스트]]
```
**예시:** `[[나의 첫 번째 노트|여기를 클릭하세요]]`는 "여기를 클릭하세요"로 표시되지만 "나의 첫 번째 노트"로 연결됩니다.

### 제목으로 링크
```markdown
[[노트 이름#제목 이름]]
```
**예시:** `[[나의 첫 번째 노트#시작하기]]`는 "시작하기" 제목으로 바로 이동합니다.

### 블록으로 링크
먼저 블록 ID를 추가합니다:
```markdown
이것은 중요한 문단입니다. ^중요블록
```
그런 다음 링크합니다:
```markdown
[[나의 첫 번째 노트#^중요블록]]
```

### 외부 링크
```markdown
[구글](https://www.google.com)
```

### 백링크(역링크)
- **백링크 패널**: 오른쪽 사이드바에서 **백링크 아이콘**을 클릭하면 현재 노트를 링크하고 있는 모든 노트를 볼 수 있습니다.
- 연결되지 않은 언급: 명시적인 `[[링크]]` 없이도 노트 이름이 언급된 곳을 보여줍니다.

**예시:** `노트 A`에 `[[노트 B]]`가 포함되어 있으면, `노트 B`를 열었을 때 백링크 패널에 `노트 A`가 표시됩니다.

### 별칭(Aliases)
노트 속성에서 여러 이름을 지정할 수 있습니다:
```yaml
---
aliases:
  - 인공지능
  - 머신러닝
---
```
이제 `[[인공지능]]`과 `[[머신러닝]]` 모두 이 노트로 연결됩니다.

## 4. 태그

태그는 노트를 분류하고 찾는 데 도움을 줍니다.

### 기본 태그
```markdown
#생산성
#학습/옵시디언
#프로젝트/진행중
```

### 중첩 태그
```markdown
#책/소설
#책/논픽션/과학
```
중첩 태그는 계층 구조를 만듭니다. `#책`을 검색하면 `#책`, `#책/소설` 등이 모두 검색됩니다.

### 태그 사용 위치
- **인라인**: 노트 본문 어디에나 태그 작성 가능
- **프론트매터**: 속성 섹션에 추가
```yaml
---
tags:
  - 생산성
  - 옵시디언
---
```

### 태그 패널
- 오른쪽 사이드바에서 **태그 패널**을 열면 볼트의 모든 태그를 볼 수 있습니다
- 태그를 클릭하면 해당 태그가 있는 노트를 필터링합니다

## 5. 폴더 & 파일 관리

### 폴더 만들기
1. **파일 탐색기**(왼쪽 사이드바)에서 우클릭
2. **"새 폴더"** 선택
3. 폴더 이름 입력

### 노트 이동
- 파일 탐색기에서 **드래그 앤 드롭**
- 우클릭 → **"파일 이동..."**

### 노트 이름 변경
- 우클릭 → **"이름 변경"** 또는 파일명 클릭 후 `F2`
- 이름이 변경된 노트로의 모든 링크가 자동으로 업데이트됩니다!

### 파일 정렬
- 파일 탐색기 상단의 정렬 아이콘을 클릭
- 옵션: 파일명(A→Z), 수정 시간, 생성 시간

### 폴더 구조 예시
```
📁 나의 볼트
├── 📁 프로젝트
│   ├── 📁 진행중
│   └── 📁 보관
├── 📁 영역
│   ├── 📁 건강
│   └── 📁 재정
├── 📁 자료
│   └── 📁 책
└── 📁 데일리 노트
```

## 6. 검색

### 기본 검색
`Ctrl/Cmd + Shift + F`를 눌러 전체 검색을 엽니다.

### 검색 연산자

| 연산자 | 예시 | 검색 결과 |
|--------|------|----------|
| `""` | `"정확한 문구"` | 정확한 문구가 포함된 노트 |
| `path:` | `path:프로젝트` | 프로젝트 폴더 내의 노트 |
| `file:` | `file:회의` | 파일명에 "회의"가 포함된 노트 |
| `tag:` | `tag:중요` | #중요 태그가 있는 노트 |
| `line:` | `line:(업무 마감)` | 같은 줄에 두 단어가 있는 노트 |
| `section:` | `section:(## 요약)` | 특정 제목 아래의 내용 |
| `block:` | `block:(아이디어 중요)` | 같은 블록/문단에 두 단어 |
| `-` | `-tag:보관` | #보관 태그가 없는 노트 제외 |
| `OR` | `고양이 OR 강아지` | 둘 중 하나가 포함된 노트 |

### 검색 예시
```
tag:프로젝트 path:진행중 -tag:완료
```
`진행중` 폴더에서 `#프로젝트` 태그가 있고 `#완료` 태그가 **없는** 모든 노트를 찾습니다.

```
"회의록" file:2024
```
파일명에 "2024"가 있고 내용에 "회의록"이 포함된 노트를 찾습니다.

## 7. 그래프 뷰

그래프 뷰는 모든 노트와 연결 관계를 **시각적 지도**로 보여줍니다.

### 그래프 뷰 열기
- 왼쪽 사이드바에서 **그래프 뷰 아이콘**을 클릭
- 또는 커맨드 팔레트: `Ctrl/Cmd + P` → "그래프 뷰" 입력

### 조작법
- **확대/축소**: 마우스 스크롤
- **이동**: 배경을 클릭하고 드래그
- **포커스**: 노드를 클릭하면 연결 관계가 강조 표시
- **노트 열기**: 노드를 더블클릭

### 필터
그래프 뷰 설정(톱니바퀴 아이콘)에서:
- **검색 필터**: 조건에 맞는 노트만 표시 (예: `path:프로젝트`)
- **태그**: 태그별로 노드에 색상 지정
- **첨부 파일**: 첨부 파일 표시/숨기기
- **기존 파일만**: 아직 만들어지지 않은 노트 링크 숨기기
- **고아 노트**: 링크가 없는 노트 표시/숨기기

### 로컬 그래프
- 노트를 열고 "추가 옵션" 메뉴(⋮)에서 **로컬 그래프**를 엽니다
- 해당 노트의 직접적인 연결만 보여줍니다

### 활용 예시
`[[파이썬]]`, `[[데이터 과학]]`, `[[머신러닝]]` 노트가 서로 링크되어 있다면, 그래프 뷰에서 하나의 클러스터(묶음)로 표시됩니다 — 이 주제들이 지식 베이스에서 밀접하게 관련되어 있음을 보여줍니다.

## 8. 캔버스

캔버스는 옵시디언의 **무한 화이트보드**로, 시각적 사고에 활용됩니다.

### 캔버스 만들기
1. `Ctrl/Cmd + P` → "새 캔버스 만들기"
2. 또는 폴더 우클릭 → "새 캔버스"

### 캔버스 요소

**카드(텍스트 카드):**
- 빈 공간을 더블클릭하면 카드가 생성됩니다
- 카드 안에 마크다운을 작성할 수 있습니다

**노트(임베드된 노트):**
- 파일 탐색기에서 노트를 캔버스로 드래그합니다
- 노트 내용이 실시간으로 표시됩니다

**미디어:**
- 이미지, PDF 등의 파일을 캔버스에 드래그합니다

**웹 링크:**
- URL을 붙여넣으면 웹 페이지가 임베드됩니다

### 연결선
- 카드 가장자리에 마우스를 올리면 화살표를 다른 카드로 드래그할 수 있습니다
- 시각적 연결/화살표가 생성됩니다

### 캔버스 레이아웃 예시
```
[연구 주제] ──→ [핵심 발견] ──→ [실행 항목]
      │                │
      ▼                ▼
[출처 노트]       [탐구할 질문]
```

## 9. 템플릿

템플릿을 사용하면 **미리 만들어둔 내용**을 노트에 삽입할 수 있습니다.

### 템플릿 설정하기
1. `Templates`(또는 원하는 이름) 폴더를 만듭니다
2. **설정 → 코어 플러그인 → 템플릿**으로 이동합니다
3. **템플릿 폴더 위치**를 해당 폴더로 설정합니다

### 템플릿 만들기
Templates 폴더에 노트를 만듭니다:

**예시: 회의록 템플릿**
```markdown
## 회의: {{title}}
**날짜:** {{date}}
**시간:** {{time}}
**참석자:**
- 

## 안건
1. 

## 논의 내용


## 실행 항목
- [ ] 

## 다음 단계

```

### 템플릿 변수
| 변수 | 출력 예시 |
|------|----------|
| `{{title}}` | 노트의 제목 |
| `{{date}}` | 오늘 날짜 (예: 2024-01-15) |
| `{{time}}` | 현재 시간 (예: 14:30) |
| `{{date:YYYY-MM-DD}}` | 사용자 지정 날짜 형식 |
| `{{time:HH:mm}}` | 사용자 지정 시간 형식 |

### 템플릿 삽입하기
1. 템플릿을 넣을 노트를 엽니다
2. `Ctrl/Cmd + P` → "템플릿 삽입"
3. 원하는 템플릿을 선택합니다
4. 변수가 치환된 내용이 삽입됩니다

## 10. 데일리 노트

데일리 노트는 매일 **새로운 노트를 자동으로** 생성합니다.

### 데일리 노트 활성화
1. **설정 → 코어 플러그인 → 데일리 노트** → 켜기(ON)

### 설정
- **날짜 형식**: `YYYY-MM-DD` (기본값) 또는 사용자 지정 (예: `YYYY/MM/MMMM/YYYY-MM-DD`)
- **새 파일 위치**: 폴더 설정 (예: `데일리 노트`)
- **템플릿**: 새 데일리 노트에 사용할 템플릿 선택

### 데일리 노트 템플릿 예시
```markdown
# {{date:dddd, YYYY년 M월 D일}}

## 🎯 오늘의 목표
- [ ] 
- [ ] 
- [ ] 

## 📝 메모


## 📖 오늘 배운 것


## 🙏 감사한 것
1. 
2. 
3. 

## ✅ 하루 마무리 회고
- 잘한 점은?
- 개선할 점은?
```

### 오늘의 데일리 노트 열기
- 왼쪽 사이드바에서 **달력 아이콘**을 클릭
- 또는 `Ctrl/Cmd + P` → "오늘의 데일리 노트 열기"

## 11. 아웃라인

아웃라인 패널은 현재 노트의 제목을 기반으로 **목차**를 보여줍니다.

### 아웃라인 열기
- 오른쪽 사이드바에서 **아웃라인 아이콘**을 클릭
- 또는 `Ctrl/Cmd + P` → "아웃라인 표시"

### 사용법
- 아무 제목을 클릭하면 해당 섹션으로 이동합니다
- 모든 제목 레벨(`#`부터 `######`까지)에서 작동합니다

### 예시
노트에 다음과 같은 내용이 있다면:
```markdown
# 서론
## 배경
## 문제 정의
# 방법론
## 데이터 수집
## 분석
# 결과
# 결론
```
아웃라인 패널에 클릭 가능한 중첩 트리가 표시됩니다.

## 15. 속성(프론트매터 / YAML)

속성은 YAML 프론트매터를 사용하여 노트에 **구조화된 메타데이터**를 추가합니다.

### 기본 문법
```yaml
---
title: 나의 연구 논문
date: 2024-01-15
status: 진행중
tags:
  - 연구
  - 과학
category: 학술
rating: 4
---
```

### 속성 유형
| 유형 | 예시 |
|------|------|
| 텍스트 | `author: 홍길동` |
| 숫자 | `rating: 5` |
| 날짜 | `created: 2024-01-15` |
| 체크박스 | `published: true` |
| 리스트 | `tags: [a, b, c]` |
| 링크 | `related: "[[다른 노트]]"` |

### 속성 활용
- 속성은 각 노트 상단의 **속성 뷰**에 표시됩니다
- 속성으로 검색: 검색에서 `[property:value]` 사용
- 커뮤니티 플러그인(Dataview 등)으로 속성을 강력하게 쿼리할 수 있습니다

### 예시: 속성이 있는 책 노트
```yaml
---
title: 아토믹 해빗
author: 제임스 클리어
rating: 5
status: 완독
date-read: 2024-01-10
tags:
  - 책
  - 자기계발
  - 습관
---

# 아토믹 해빗 - 노트

## 핵심 내용
- 습관은 자기 계발의 복리 이자다
- 목표가 아닌 시스템에 집중하라
```

## 16. 콜아웃

콜아웃은 중요한 정보를 강조하기 위한 **스타일이 적용된 블록**입니다.

### 기본 문법
```markdown
> [!note]
> 이것은 노트 콜아웃입니다. 일반 정보에 사용합니다.

> [!tip]
> 유용한 팁입니다!

> [!warning]
> 이것에 주의하세요!

> [!danger]
> 중요한 경고 — 무시하지 마세요!
```

### 모든 콜아웃 유형
| 유형 | 아이콘 | 용도 |
|------|--------|------|
| `note` | 📝 | 일반 정보 |
| `abstract` / `summary` | 📋 | 요약 |
| `info` | ℹ️ | 정보성 |
| `tip` / `hint` | 🔥 | 유용한 팁 |
| `success` / `check` | ✅ | 성공/긍정적 결과 |
| `question` / `faq` | ❓ | 질문 |
| `warning` / `caution` | ⚠️ | 경고 |
| `failure` / `fail` | ❌ | 실패 |
| `danger` / `error` | 🚫 | 심각한 경고 |
| `bug` | 🐛 | 버그 리포트 |
| `example` | 📖 | 예시 |
| `quote` / `cite` | 💬 | 인용 |

### 사용자 지정 제목
```markdown
> [!tip] 나만의 제목
> 내용을 여기에 작성합니다.
```

### 접을 수 있는 콜아웃
```markdown
> [!faq]- 클릭하면 펼쳐집니다 (기본: 접힌 상태)
> 이 내용은 클릭할 때까지 숨겨져 있습니다.

> [!faq]+ 클릭하면 접힙니다 (기본: 펼쳐진 상태)
> 이 내용은 기본적으로 표시됩니다.
```

### 중첩 콜아웃
```markdown
> [!question] 콜아웃을 중첩할 수 있나요?
> > [!success] 네, 가능합니다!
> > 이렇게 말이죠.
```

## 17. 임베드 & 트랜스클루전

다른 노트의 내용을 **현재 노트에 직접 삽입**합니다.

### 전체 노트 임베드
```markdown
![[노트 이름]]
```

### 제목 섹션 임베드
```markdown
![[노트 이름#제목]]
```

### 블록 임베드
```markdown
![[노트 이름#^블록-id]]
```

### 이미지 임베드
```markdown
![[이미지.png]]
![[이미지.png|400]]  <!-- 너비 지정 -->
```

### PDF 임베드
```markdown
![[문서.pdf]]
![[문서.pdf#page=5]]  <!-- 특정 페이지 -->
```

### 오디오/비디오 임베드
```markdown
![[녹음.mp3]]
![[영상.mp4]]
```

### 예시: 임베드를 활용한 대시보드 노트
```markdown
# 나의 대시보드

## 현재 프로젝트
![[진행중 프로젝트#진행중]]

## 오늘의 할 일
![[2024-01-15#할 일]]

## 빠른 참조
![[단축키 모음#자주 쓰는 것]]
```

## 18. 화면 분할 & 워크스페이스

### 화면 분할
- **오른쪽 분할**: 탭 우클릭 → "오른쪽으로 분할"
- **아래쪽 분할**: 탭 우클릭 → "아래쪽으로 분할"
- **탭 드래그**: 탭을 아무 가장자리로 드래그하여 분할

### 탭 그룹
- 각 창에 여러 탭을 둘 수 있습니다
- 탭을 창 그룹 간에 드래그할 수 있습니다

### 연결된 뷰
- 탭 우클릭 → "연결된 뷰 열기" → "백링크", "아웃라인", "로컬 그래프" 선택
- 연결된 뷰는 해당 창의 활성 노트를 따라갑니다

### 워크스페이스
**다양한 화면 레이아웃**을 저장하고 불러올 수 있습니다:
1. **워크스페이스** 코어 플러그인을 활성화합니다
2. `Ctrl/Cmd + P` → "워크스페이스 관리"
3. 현재 레이아웃을 이름으로 저장합니다 (예: "글쓰기", "리서치", "리뷰")
4. 저장된 워크스페이스 간에 전환합니다

### 워크스페이스 예시: 리서치
```
┌─────────────────┬─────────────────┐
│  참고 자료       │  나의 노트       │
│  (읽기)          │  (쓰기)         │
├─────────────────┼─────────────────┤
│  그래프 뷰       │  아웃라인        │
│  (로컬)          │  (네비게이션)    │
└─────────────────┴─────────────────┘
```

| **Periodic Notes** | 주간/월간/연간 노트 |
| **Advanced Tables** | 향상된 테이블 편집 |
| **Linter** | 노트 자동 포맷팅 |
| **Iconize** | 폴더와 파일에 아이콘 추가 |

### 플러그인 설치하기
1. **설정 → 커뮤니티 플러그인 → 탐색**

3. **설치**를 클릭합니다
4. **활성화**를 클릭합니다
5. **설정 → 커뮤니티 플러그인 → (플러그인 이름)**에서 설정합니다

### Dataview 예시
Dataview를 설치한 후 노트를 쿼리할 수 있습니다:
````markdown
```dataview
TABLE author, rating, status
FROM #책
SORT rating DESC
```
````
이것은 `#책` 태그가 있는 모든 노트의 저자, 평점, 상태를 테이블로 만듭니다.

## 30. 팁 & 베스트 프랙티스

### 초보자를 위한 조언
1. **간단하게 시작하세요**: 처음부터 복잡한 시스템을 구축하려 하지 마세요
2. **먼저 쓰고, 나중에 정리하세요**: 아이디어를 포착하는 것에 집중하세요
3. **적극적으로 링크하세요**: 한 노트가 다른 노트와 관련될 때마다 `[[링크]]`를 만드세요
4. **데일리 노트를 활용하세요**: 일기 쓰기 습관을 만드세요
5. **한 번에 하나의 기능만 배우세요**: 자신을 압도하지 마세요

### 노트 정리 방법론
| 방법론 | 설명 | 적합한 용도 |
|--------|------|------------|
| **제텔카스텐** | 링크로 연결된 원자적 노트 | 연구, 지식 축적 |
| **PARA** | 프로젝트, 영역, 자료, 보관 폴더 | 작업 관리 |
| **MOC** | 관련 노트로의 링크 모음인 인덱스 노트 | 탐색, 개요 |
| **데일리 노트 + 링크** | 주제 노트로 링크가 있는 일일 저널 | 저널링, 기록 |

### MOC(Map of Content) 예시
```markdown
# 프로그래밍 MOC

## 언어
- [[파이썬]]
- [[자바스크립트]]
- [[러스트]]

## 개념
- [[자료구조]]
- [[알고리즘]]
- [[디자인 패턴]]

## 프로젝트
- [[웹 앱 프로젝트]]
- [[CLI 도구 프로젝트]]
```

### 유용한 트릭
- **호버 미리보기**: `Ctrl/Cmd`을 누른 채 링크 위에 마우스를 올리면 노트 미리보기가 표시됩니다
- **중요한 노트를 즐겨찾기**: 빠른 접근을 위해 북마크를 사용하세요
- **일관된 제목 사용**: 아웃라인 탐색과 임베드를 쉽게 만듭니다
- **홈 노트 만들기**: 주요 영역으로 연결되는 시작점 대시보드
- **정기적인 검토**: 주기적으로 고아 노트를 검토하고 링크하세요
